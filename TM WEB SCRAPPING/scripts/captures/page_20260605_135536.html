SEC · 04 / METODOSEC · 04 / METODOX·04
harness
O Método

Método > ferramenta.

Seis camadas. Um harness. Ferramenta substituível — aqui é o sistema:

01

Contexto antes de prompt

Estruturar o que o modelo precisa saber antes de gerar — fugindo da tentativa e erro.

02

Engenharia de contexto

Organizar contexto pro LLM produzir código que você não precisa reescrever. O padrão .context/ que o mercado ainda vai descobrir.

03

Spec-driven development

Definir o que tem que existir antes de gerar código. Validar que existiu depois.

04 · ATIVO

Harness engineering

A base que envolve o modelo: contexto, spec e validation loop. O que permanece quando a ferramenta troca.

05

PREVC workflow

Plan → Review → Execute → Validate → Confirm. O ciclo que separa quem usa LLM como copiloto de quem usa como muleta.

06

Validation loop

Auditar output, medir drift e fechar o ciclo antes do merge — engenharia, não vibe.
