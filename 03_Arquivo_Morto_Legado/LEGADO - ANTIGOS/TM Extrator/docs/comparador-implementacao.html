1. Visao Geral do Comparador

O repositorio implementa extracao de itens de DOCX e consolidacao de resultados.

  1. O produto principal e um extrator de itens de tabelas em arquivos .docx com interface web em React/Vite e processamento local no navegador. README.md src/App.jsx
  2. Existe um fluxo alternativo via script local em Python com GUI para selecionar o DOCX, gerar Excel e log. extrair_itens_docx.py
  3. O termo "comparador" nao aparece no codigo, mas a funcionalidade mais proxima e a consolidacao/agrupamento de itens repetidos. src/App.jsx extrair_itens_docx.py
Nota de arquitetura: existem duas implementacoes da mesma logica central (web e Python), ambas filtrando tabelas com cabecalho "Itens" e gerando saidas em Excel/log. src/App.jsx extrair_itens_docx.py
  • Extracao de tabelas "Itens" a partir de DOCX. src/App.jsx extrair_itens_docx.py
  • Geracao de planilha Excel com itens extraidos. src/App.jsx extrair_itens_docx.py
  • Geracao de log textual da execucao. src/App.jsx extrair_itens_docx.py
  • Consolidacao/agrupamento de itens repetidos. src/App.jsx extrair_itens_docx.py

2. Stack e Arquitetura (pastas, entrypoints, camadas)

Resumo de tecnologias e organizacao do repositorio.

  1. Frontend: React + Vite com entrada HTML em index.html que aponta para src/main.jsx. package.json vite.config.js
  2. Processamento no navegador: JSZip (leitura do DOCX), XLSX (geracao de planilhas), framer-motion e lucide-react para UI. package.json src/App.jsx
  3. Script local: Python com python-docx, openpyxl e Tkinter. extrair_itens_docx.py
  4. Camadas e pastas relevantes: UI e logica principal em src/; assets em public/; build em dist/; referencias de design em exemplo do gpt/; guias de design em Agents_design-v2/ e Agents_design-v3/. README.md
  • Entrypoint web: index.html -> src/main.jsx -> src/App.jsx.
  • Entrypoint Python: extrair_itens_docx.py (tambem via rodar.bat).
  • Configuracao Vite: vite.config.js.
  • Linting React/JS: eslint.config.js.

3. Fluxo de Funcionamento (entrada -> processamento -> saida)

Fluxos detalhados para web e Python.

3.1 Fluxo Web (React)

  1. Entrada: usuario seleciona um arquivo .docx e o app valida a extensao. src/App.jsx
  2. Processamento: o DOCX e aberto via JSZip, o XML e parseado, as tabelas com cabecalho "Itens" sao filtradas e as linhas validas sao extraidas. src/App.jsx
  3. Saida: planilha Excel e log sao gerados e disponibilizados para download; opcionalmente ocorre consolidacao por regra. src/App.jsx
  • Leitura do DOCX pelo navegador. src/App.jsx
  • Regra de validacao de codigo e quantidade. src/App.jsx
  • Downloads (Excel e log). src/App.jsx

3.2 Fluxo Python (GUI)

  1. Entrada: selecao do DOCX e do caminho de saida via Tkinter. extrair_itens_docx.py
  2. Processamento: leitura das tabelas com python-docx, filtro de cabecalho "Itens" e extracao de codigo/quantidade. extrair_itens_docx.py
  3. Saida: geracao de Excel com abas "Itens" e "Consolidado" e log em arquivo texto. extrair_itens_docx.py
  • Planilha principal com itens extraidos. extrair_itens_docx.py
  • Planilha consolidada por codigo. extrair_itens_docx.py
  • Log detalhado com linhas ignoradas. extrair_itens_docx.py

4. Componentes Principais e Responsabilidades

Funcoes e modulos responsaveis pelo "comparador" (consolidacao).

  1. extractItemsFromDocx faz a leitura do DOCX no navegador e extrai itens validos. src/App.jsx
  2. aggregateItems consolida itens por regra (codigo, descricao, ou ambos). src/App.jsx
  3. buildXlsx cria a planilha Excel e dispara o download. src/App.jsx
  4. buildLogText gera o log de execucao. src/App.jsx
  5. extract_code_qty, consolidate_rows e save_xlsx implementam o fluxo equivalente no Python. extrair_itens_docx.py

const ag = aggregateItems(items, aggRule);

6. Como Estender o Comparador (novos criterios/fontes/regras)

Locais exatos no codigo onde as regras podem ser alteradas.

  1. Para novas regras de agrupamento no web, ajuste aggregateItems e a lista de opcoes de regra. src/App.jsx
  2. Para novas regras no Python, altere consolidate_rows e como a chave e calculada. extrair_itens_docx.py
  3. Para novas fontes ou criterios de identificacao de tabela, ajuste o filtro de cabecalho "Itens". src/App.jsx extrair_itens_docx.py
  4. Para alterar leitura de quantidade, modifique pickQuantityFromRow / pick_quantity_from_row. src/App.jsx extrair_itens_docx.py
Dica: mantenha as regras alinhadas entre o web e o Python para resultados consistentes. src/App.jsx extrair_itens_docx.py
  • Atualizar UI das regras no web. src/App.jsx
  • Atualizar logica de consolidacao no Python. extrair_itens_docx.py
  • Revisar logs para validar novas regras. log_summary.txt output.txt

7. Riscos Reais Identificados e Mitigacoes

Riscos baseados em artefatos reais do repositorio.

  1. Risco: consolidacao gera itens duplicados quando ha variacao de caracteres Unicode na descricao (ex.: hifen diferente), reproduzido em logs e script de reproducao. log_summary.txt output.txt reproduce_issue.js src/App.jsx
  2. Mitigacao sugerida: normalizar Unicode antes de agrupar (ex.: remover variacoes de hifen). src/App.jsx extrair_itens_docx.py
  3. Risco: apenas tabelas cujo cabecalho e exatamente "Itens" sao processadas, o que pode ignorar variacoes de nome. src/App.jsx extrair_itens_docx.py
  4. Mitigacao sugerida: ampliar a regra de deteccao para aceitar sinonimos ou colunas. src/App.jsx extrair_itens_docx.py
  5. Risco: o texto "Max: 20MB" esta na UI, mas nao ha validacao de tamanho de arquivo no codigo. src/App.jsx
  6. Mitigacao sugerida: adicionar validacao de tamanho antes de processar. src/App.jsx
  7. Risco: o script Python depende de GUI (Tkinter) e pode falhar em ambientes sem interface grafica. extrair_itens_docx.py
  8. Mitigacao sugerida: criar modo CLI sem GUI. extrair_itens_docx.py
Atencao: o risco de duplicidade em consolidacao ja foi reproduzido, portanto merece prioridade. log_summary.txt output.txt
Seguranca: o fluxo web processa arquivos localmente; nao ha referencias a envio de dados para servidor no codigo analisado. README.md src/App.jsx
  • Normalizacao de texto antes de agrupar. src/App.jsx extrair_itens_docx.py
  • Expansao da regra de identificacao de tabelas. src/App.jsx extrair_itens_docx.py
  • Validacao de tamanho de arquivo no upload. src/App.jsx
  • Modo CLI para ambientes sem GUI. extrair_itens_docx.py
