§01 Contexto da Skill

O que é o UI UX Pro Max

O ui-ux-pro-max é um banco de dados de inteligência de design para IAs. Ele contém 57 estilos visuais, 95 paletas de cores, 56 combinações tipográficas, 24 tipos de gráficos, diretrizes para 8 stacks e 98 regras de UX.

No Antigravity, a skill está em .agent/workflows/ui-ux-pro-max.md e é ativada automaticamente com o comando /ui-ux-pro-max ou quando você faz tarefas de UI. Ela funciona como base de referência: a IA pesquisa internamente o estilo, paleta e tipografia mais adequados ao contexto antes de gerar código.

§02 Fluxo de Trabalho

Como usar no Antigravity

O fluxo tem duas fases: instalação única no projeto e uso cotidiano. A skill é um arquivo Markdown que a IA lê como contexto de cada tarefa de UI.

Fase 1 — Instalação (uma vez por projeto)

01
Instalar o CLI globalmente

Roda uma vez na máquina. Requer Node.js instalado.

npm install -g uipro-cli
uma vez na máquina
02
Navegar até a pasta do projeto

O comando de init cria os arquivos dentro do projeto atual.

cd /caminho/do/seu/projeto
# Exemplo:
cd ~/projetos/TM-Hub
uma vez por projeto
03
Inicializar a skill para Antigravity

Cria a pasta .agent/workflows/ com o workflow da skill dentro do projeto.

uipro init --ai antigravity
uma vez por projeto
04
Copiar este README para a raiz do projeto

Coloque este arquivo (README-TM-UIUX.html) na raiz do projeto. O Antigravity pode ser instruído a ler este documento como contexto de design ao iniciar qualquer tarefa de UI.

uma vez por projeto

§03 Tokens do Design System

Ocean Breeze — Variáveis CSS Canônicas

Todos os projetos TM usam estas variáveis. A IA deve sempre usar os tokens abaixo ao gerar CSS — nunca hardcodar valores hexadecimais diretamente. O sistema usa dual-layer: --TM- para tokens canônicos e --tm- para aliases de contexto. 

/* ── Marca ─────────────────────────────── */
--TM-primary:        #22c55e;   /* verde principal */
--TM-primary-hover:  #16a34a;
--TM-primary-light:  #dcfce7;
--TM-gradient:       linear-gradient(135deg, #22c55e 0%, #06b6d4 100%);

/* ── Tipografia ─────────────────────────── */
--TM-font-sans:      "DM Sans", sans-serif;
--TM-font-serif:     "Lora", serif;
--TM-font-mono:      "IBM Plex Mono", monospace;

/* ── Sombras (light mode com tint verde) ── */
--TM-shadow-sm:  0px 4px 8px -1px rgba(34,197,94,0.08);
--TM-shadow-md:  0px 8px 16px -2px rgba(34,197,94,0.10);
--TM-shadow-lg:  0px 20px 40px -4px rgba(34,197,94,0.12);

/* ── Raios de borda ─────────────────────── */
--TM-radius-sm:  4px;
--TM-radius-md:  8px;
--TM-radius-lg:  12px;
--TM-radius-xl:  16px;
/* Sobrescrever apenas os aliases --tm- no dark mode */
html.dark {
  --tm-bg:          #0f172a;
  --tm-bg-card:     #1e293b;
  --tm-text:        #d1d5db;
  --tm-text-muted:  #a1a1aa;
  --tm-primary:     #34d399;  /* verde mais suave no dark */
  --tm-border:      #4b5563;
}
⚠️ Nunca use #22c55e direto no CSS. Sempre referencie via var(--TM-primary) ou var(--tm-primary). Isso garante que o dark mode funcione automaticamente.
Token Uso correto Nunca usar
Cor de ação primária var(--TM-primary) #22c55e direto
Botão primário (fundo) var(--TM-gradient) ou var(--TM-primary-hover) var(--TM-primary) — contraste insuficiente com texto branco
Texto principal var(--tm-text) #0f172a direto
Background de card var(--tm-bg-card) #ffffff direto
Sombra de hover var(--TM-shadow-md) box-shadow hardcoded
§04 Estilo Visual

Identidade Ocean Breeze

O Ocean Breeze é um estilo Clean Professional com influências de Dashboard Analytics. É o design system de todas as aplicações internas TM. Ao gerar qualquer interface, a IA deve respeitar estas características:

Dimensão Decisão TM Racional
Estilo geral Clean Professional · Dashboard Equipe técnica de engenharia, uso diário intenso
Cor primária Verde #22c55e → gradiente com ciano Identidade TM, reforça sucesso/confirmação
Tipografia display Lora (serif italic) Personalidade refinada em títulos e hero
Tipografia UI DM Sans Alta legibilidade em tamanhos pequenos (labels, nav)
Tipografia técnica IBM Plex Mono Badges, status, valores numéricos, código
Ícones SVG — Lucide Icons ou Phosphor Herdam currentColor, funcionam no dark mode
Emojis Apenas em microcopy informal (toasts, empty states) Renderização inconsistente entre SOs; tom inadequado para engenharia
Animações Sutis, com propósito (stagger, shimmer, hover) Nunca animações longas ou distrativas
Dark mode html.dark — tokens --tm- sobrescritos Persistência via localStorage
Densidade Media-alta — ferramenta interna, não landing page Equipe usa o sistema horas por dia
Layout de Documentação Modelo "Pages": Header fixo, nav central <div class="container"> Páginas standalone como commit.html e catálogos devem seguir esse grid.
💡 Contexto de produto: os apps TM são usados por uma equipe de relatórios de engenharia. O tom visual deve transmitir precisão técnica e confiabilidade — não leveza ou entretenimento. Prefira densidade de informação a espaço em branco excessivo.
§05 Regras UX Obrigatórias

O que a IA deve sempre fazer

Estas regras se aplicam a qualquer componente gerado para projetos TM:

Regra Como implementar
Foco visível :focus-visible { outline: 2px solid var(--TM-primary); outline-offset: 2px; }
Contraste de botão Fundo do .btn-primary = var(--TM-primary-hover) (#16a34a), não a primária pura
Elementos interativos Spans e divs clicáveis precisam de role="button" e tabindex="0"
Ícone sem texto visível Sempre adicionar aria-label="descrição da ação"
Tamanho mínimo de fonte Nenhum texto interativo abaixo de 0.75rem (12px)
Empty states Listas e grids vazios sempre exibem mensagem — nunca espaço em branco silencioso
Redução de movimento Incluir @media (prefers-reduced-motion: reduce) em componentes com animação
Responsividade mínima Stats grids 4 col → 2 col abaixo de 900px → 1 col abaixo de 600px

Anti-patterns — nunca fazer

/* Texto branco em verde puro — contraste insuficiente (3.1:1) */
.btn { background: #22c55e; color: white; }

/* Cor hardcoded — quebra o dark mode */
.card { background: #ffffff; color: #0f172a; }

/* Emoji como ícone de navegação */
<span class="nav-icon">📊</span>

/* Span clicável sem role */
<span onclick="filter()">Todos</span>
/* Contraste adequado com primary-hover */
.btn { background: var(--TM-primary-hover); color: white; }

/* Tokens — dark mode automático */
.card { background: var(--tm-bg-card); color: var(--tm-text); }

/* SVG com currentColor */
<svg aria-hidden="true">...</svg> + texto visível ou aria-label

/* Role correto */
<span role="button" tabindex="0" onclick="filter()">Todos</span>
