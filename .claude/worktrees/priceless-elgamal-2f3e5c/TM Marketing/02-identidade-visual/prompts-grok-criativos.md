# Prompts para Grok — Criação de Conteúdo TM Sempre Tecnologia

**Como usar:** Abra o Grok (grok.com ou app X), cole o prompt exatamente como está, substitua os campos entre [colchetes] e execute. Cada prompt está otimizado para o formato e canal indicado.

---

## PROMPT BASE DE CONTEXTO
> Cole este contexto no início de qualquer sessão nova com o Grok antes de usar os prompts abaixo. Isso evita ter que repetir toda vez.

```
Você vai me ajudar a criar conteúdo de marketing para a TM Sempre Tecnologia.

CONTEXTO DA EMPRESA:
- Nome: TM Sempre Tecnologia
- Fundador: Thiago Nascimento Barbosa
- Localização: Goiânia, GO — Brasil
- Produto principal: AutoRelatório V2 — software que automatiza a geração de relatórios fotográficos de vistoria predial (contratos Banco do Brasil / Caixa Econômica Federal). Reduz de 4 horas para 15 minutos por relatório.
- Serviços: desenvolvimento web, mobile, automação de processos, consultoria técnica
- Público: gestores e engenheiros de empresas de manutenção predial, donos de PMEs que precisam de tecnologia

VOZ DA MARCA:
- Tom: técnico, direto, confiante — como um engenheiro sênior que resolve problemas
- Sem jargão de marketing ("disruptivo", "sinergia", "mindset")
- Frases curtas, objetivas, com dados reais quando possível
- Nunca promessas vagas — sempre benefício concreto
- NÃO somos influencers — somos especialistas

IDENTIDADE VISUAL (para prompts de imagem):
- Fundo escuro: #161513 (quase preto, tom quente)
- Cor de destaque: #E47A4A (laranja construção)
- Tipografia display: Roboto Slab 700
- Tipografia corpo: Inter 400
- Sem gradientes, sem cores vibrantes extras
- Grid técnico de fundo (linhas finas em baixa opacidade)
- Ícones estilo Lucide (traço fino, sem preenchimento)
```

## PROMPTS DE TEXTO (Copy)

### PT-01 — Post LinkedIn: Autoridade Técnica
```
Escreva um post para LinkedIn seguindo exatamente esta estrutura:

TEMA: [descreva o tema, ex: "como automatizamos a geração de relatórios Word com Python"]

ESTRUTURA OBRIGATÓRIA:
- Linha 1 (gancho): afirmação ou dado concreto que para o scroll — máximo 12 palavras
- Linhas 2-4: contexto do problema real
- Linha em branco
- Desenvolvimento em parágrafos curtos (máximo 2 linhas cada)
- Linha em branco
- Conclusão prática: 1 coisa que o leitor pode fazer agora
- CTA suave: pergunta ou convite para comentar
- 4 hashtags específicas: #automação #desenvolvimentosoftware + 2 do tema

REGRAS:
- Tom técnico e direto, como engenheiro sênior
- Sem ponto de exclamação
- Sem "Hoje vim falar sobre..."
- Sem jargão de marketing
- Máximo 1200 caracteres no total
- NÃO incluir link no corpo do post (vai no comentário)

VOZ: Thiago fala em primeira pessoa como fundador e especialista.
```

### PT-02 — Post LinkedIn: Case / Resultado
```
Escreva um post LinkedIn sobre um resultado real usando esta estrutura:

DADOS DO CASE:
- Segmento do cliente: [ex: empresa de manutenção predial em Goiás]
- Problema antes: [ex: 4 horas montando relatórios de vistoria manualmente]
- Solução: [ex: AutoRelatório V2]
- Resultado: [ex: 15 minutos por relatório, economizando 3h/dia por técnico]

ESTRUTURA:
Linha 1: o resultado em números (gancho)
Parágrafo 1: como era o processo antes (concreto, sem generalizar)
Parágrafo 2: o que mudou e por que funcionou
Parágrafo 3: o que isso representa na prática (horas, custo, produtividade)
Fechamento: convite sutil para quem tem problema similar

TOM: factual, sem euforia. Deixa o dado falar.
TAMANHO: 800–1000 caracteres
```

### PT-03 — Legenda Instagram: Carrossel Técnico
```
Escreva a legenda de um post carrossel para Instagram.

TEMA DO CARROSSEL: [ex: "4 processos que você ainda faz manualmente (e não deveria)"]
NÚMERO DE SLIDES: [ex: 6]

ESTRUTURA DA LEGENDA:

- 2-3 linhas de complemento ou dado adicional não mostrado nos slides
- CTA: "salva esse aqui" ou "manda pra quem precisa disso"
- 6-8 hashtags: mix de grande (#tecnologia, #automação) + nicho (#autorelatorio, #manutencaopredial)

TAMANHO: máximo 300 caracteres antes das hashtags
TOM: mais acessível que o LinkedIn, mas ainda técnico e direto
```

### PT-04 — Artigo de Blog
```
Escreva um artigo completo para o blog da TM Sempre Tecnologia.

TÍTULO SUGERIDO: [ex: "Como gerar relatórios Word automaticamente com Python e python-docx"]
PALAVRA-CHAVE PRINCIPAL: [ex: "automação de relatórios Word python"]
PÚBLICO: gestores e desenvolvedores que querem automatizar documentos
EXTENSÃO: mínimo 900 palavras

ESTRUTURA OBRIGATÓRIA:
1. Headline H1 com palavra-chave
2. Lead (2-3 linhas): problema que o artigo resolve
3. H2: O Problema (contexto real, dados se possível)
4. H2: A Solução (com exemplo de código ou passo a passo real)
5. H2: Na Prática (exemplo aplicado ao contexto do leitor)
6. H2: O Que Fazer Agora (1 ação concreta)
7. CTA final: link para contato ou produto relacionado

REGRAS:
- Exemplos de código reais e funcionais (Python se aplicável)
- Sem introdução genérica ("Neste artigo você vai aprender...")
- Dados específicos quando existirem
- Tom de especialista que já resolveu o problema
- SEO: palavra-chave no título, primeiro parágrafo e pelo menos 2 H2s
```

### PT-05 — Copy Meta Ads (Texto do Anúncio)
```
Escreva 3 variações de copy para anúncio Meta Ads.

PRODUTO: [AutoRelatório V2 / Desenvolvimento custom / Consultoria]
ETAPA DO FUNIL: [Topo — awareness / Meio — consideração / Fundo — conversão]
PÚBLICO: [ex: gestores de empresas de manutenção predial com contratos BB/Caixa]
OBJETIVO DO ANÚNCIO: [ex: mensagem no WhatsApp / tráfego para landing page]

PARA CADA VARIAÇÃO, escreva:
- Headline (máximo 40 caracteres): direto ao problema ou resultado
- Texto principal (máximo 125 caracteres): problema + solução + CTA
- Descrição (máximo 30 caracteres): reforço do benefício

REGRAS:
- Sem promessas vagas
- Sem urgência falsa ("só hoje!")
- CTA claro e único por variação
- Tom direto, sem euforia
- Testar: 1 variação focada no problema, 1 no resultado, 1 na oferta
```

### PT-06 — Script de Vídeo (Reels / YouTube Shorts)
```
Escreva um script para vídeo curto de 30-45 segundos.

TEMA: [ex: "mostrando o AutoRelatório gerando um relatório em tempo real"]
FORMATO: [screen recording / fala para câmera / bastidor]
CANAL: [Instagram Reels / YouTube Shorts]

ESTRUTURA:
- 0–3s: hook visual ou frase de abertura (o que vai ver / problema que resolve)
- 4–20s: desenvolvimento (demonstração ou explicação)
- 21–30s: resultado ou conclusão
- 31–45s (opcional): CTA ("link na bio" / "manda uma mensagem")

SAÍDA ESPERADA:
- Texto do narrador (se tiver voz)
- Texto das legendas por bloco de tempo
- Indicação de corte ou transição
- Sugestão de texto sobreposto na tela

TOM: direto, sem enrolação. Cada segundo conta.
```

## WORKFLOW DE CRIAÇÃO — Como usar tudo junto

```
SEGUNDA-FEIRA (planejamento)
  1. Abra o Grok
  2. Cole o PROMPT BASE DE CONTEXTO
  3. Use PT-04 para escrever o artigo da semana
  4. Salve o artigo em /03-blog/

TERÇA (distribuição do artigo)
  1. Use PT-01 (com o tema do artigo) → post LinkedIn
  2. Use PT-03 → legenda do carrossel Instagram (resumo do artigo)
  3. Use PI-02 → imagem de cover do artigo
  4. Publique artigo + distribua nos canais

QUARTA (conteúdo independente)
  1. Use PT-02 → case ou resultado
  2. Use PI-01 → imagem do dado principal
  3. Publique no LinkedIn + Facebook

SEXTA (processo ou reflexão)
  1. Use PT-01 com tema de bastidor/processo
  2. Publique (sem imagem necessária — texto performa bem no LinkedIn)

QUANDO PRECISAR DE AD
  1. Use PT-05 → 3 variações de copy
  2. Use PI-05 → criativo visual
  3. Suba no Meta Ads Manager
```

## Dicas de Uso com o Grok

- **Memória de contexto:** Cole o PROMPT BASE no início de cada sessão nova. O Grok não guarda entre sessões.
- **Iteração:** Se o resultado não ficou bom, adicione: *"Refaça mais direto, menos [problema específico]."*
- **Tom:** Se ficar muito formal, adicione: *"Mantenha o tom técnico mas menos corporativo — como alguém que fala de igual para igual."*
- **Dados reais:** Sempre que tiver um número real (tempo economizado, clientes atendidos, etc), coloque no prompt — o Grok vai usá-los e o conteúdo fica muito melhor.
- **Imagens:** O Grok Image funciona bem com prompts em inglês. Os prompts PI-0X já estão em inglês por isso.
