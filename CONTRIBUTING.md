# Contribuindo para o Projeto

Obrigado por considerar contribuir para o nosso projeto! Este documento fornece diretrizes e informações para ajudar você a contribuir efetivamente.

## Como Contribuir

1. Faça um fork do repositório
2. Crie uma branch para sua feature ou correção (`git checkout -b feature/nome-da-sua-feature`)
3. Faça suas alterações
4. Commit suas mudanças seguindo as [Convenções de Commit](#convenções-de-commit)
5. Push para a branch (`git push origin feature/nome-da-sua-feature`)
6. Abra um Pull Request

## Convenções de Commit

Usamos Conventional Commits para padronizar as mensagens de commit e facilitar a geração automática de changelogs.

| Prefixo | Uso |
|---------|-----|
| feat: | Nova skill, agent, guia ou funcionalidade |
| fix: | Correção de bug |
| docs: | Documentação |
| chore: | Manutenção |
| refactor: | Reestruturação sem mudança de comportamento |
| perf: | Melhoria de performance |
| test: | Testes |

### Exemplos

- `feat: adicionar skill de análise de dados`
- `fix: corrigir bug na geração de relatórios PDF`
- `docs: atualizar README com instruções de instalação`
- `chore: atualizar dependências do projeto`
- `refactor: refatorar módulo de autenticação para melhorar legibilidade`
- `perf: otimizar consulta ao banco de dados reduzindo tempo de resposta`
- `test: adicionar testes unitários para o módulo de validação`

## Diretrizes Adicionais

- Mantenha o foco nas suas alterações relacionadas ao problema que está resolvendo
- Escreva mensagens de commit claras e descritivas
- Inclua testes para novas funcionalidades sempre que possível
- Atualize a documentação quando necessário
- Siga o estilo de código existente no projeto

## Código de Conduta

Por favor, note que este projeto é lançado com um Código de Conduta. Ao participar deste projeto, você concorda em cumprir seus termos.

## Perguntas?

Se você tiver alguma dúvida, não hesite em abrir uma issue ou entrar em contato com os mantenedores do projeto.
